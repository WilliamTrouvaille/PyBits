# Python 工程标准

## 工具偏好

- 优先使用 `uv` 执行 Python、管理依赖和项目命令（当仓库支持时）。
- 优先使用 `ruff` 进行 linting 和 formatting（当已配置或可用时）。
- 新建项目代码以 Python 3.13+ 为目标，除非仓库声明了不同的支持版本。
- 假设 VSCode 为首选编辑器环境，但除非用户要求，否则不添加编辑器配置文件。
- 注释、日志和用户可见文本的语言风格跟随仓库现有惯例；无明确惯例时跟随用户语言。
- 传递目录作为参数（如 `ruff format src/`），不切换工作目录。

## 类型标注

- 类型标注是 Python 代码质量的一等公民，不是可选装饰。
- 为新的公共函数、方法和非平凡辅助函数添加参数和返回类型标注。无前导 `_` 的名称视为公共；含分支、I/O、数据转换或复用行为的辅助函数视为非平凡。
- 有意不返回值的函数标注 `-> None`。
- 仓库 Python 版本支持时，优先使用内置集合语法（`list[str]`、`dict[str, int]`）。
- 引入 PEP 695 泛型等新语法前，先遵循仓库现有类型标注风格。
- 类型迁移与修复工作流见 `typing-guide.md`（仅当本次任务涉及类型迁移时，按入口路由读取）。

## 设计偏好

- 小问题用小方案。
- 每改完一个文件后自问："资深工程师会试图简化我改动的代码吗？"如果是，先简化再继续。
- 当面向对象设计有助于厘清职责、状态或扩展点时，优先使用。
- 保持类和方法解耦，每个单元一个清晰的职责。
- 可调行为优先用配置文件；项目无更强惯例时默认使用 YAML。
- 可复用代码暴露实用的 hooks，条件是它们消除真实重复或使扩展变得自然。
- 不为了满足风格偏好而添加抽象。

## 测试

- 为改动行为添加或更新聚焦的测试。bug 修复优先用复现故障的回归测试。
- 核心逻辑保持可测试：在可行时将其与 CLI、文件 I/O、网络 I/O 和全局状态分离。
- **小改动不引入大型测试框架或重型 fixture 体系。** 使用现有测试工具和最小聚焦检查。
- 测试模式与策略见 `testing-guide.md`（仅当本次任务涉及测试时，按入口路由读取）。

## 依赖与资源卫生

- 不假设未声明的依赖存在。
- 文件读写默认使用 UTF-8 编码。
- 避免将大文件一次性读入内存。

## 日志

- 新日志优先使用 `loguru`，除非仓库已标准化于其他日志层。
- 可靠性要求：在重要操作的开始、成功完成和异常路径记录日志。
- 有意识地使用不同日志级别：`debug` 用于诊断细节，`info` 用于正常里程碑，`warning` 用于可恢复的问题，`error` 或 `exception` 用于失败。
- 异常日志保留诊断上下文，但不暴露敏感输入。
- 固定节横幅保持周围 `=` 行长度一致，固定消息居中：

  ```python
  logger.info("=" * 80)
  logger.info("初始化完成".center(80))
  logger.info(f"处理目标: '{self.target}'")
  logger.info("=" * 80)
  ```

- 不在微型脚本或低级辅助函数中强加横幅日志——那只会制造噪音。

## 注释

- 注释优先解释 Why。
- 仅当 What 澄清了非显而易见的意图、领域含义、状态转换或节导航时才添加。
- 仅当 How 涉及非显而易见的算法、协议、性能约束、兼容性变通或防御性检查时才添加。
- 不复述显而易见的代码行为、标识符或赋值。
- 文件超过 100 行时，先检查是否过度工程或应拆分。
- 较长文件有正当理由时，使用稀疏的分组注释改善导航：

  ```python
  # ========================================================================
  # 1. 初始化与构造
  # ========================================================================

  # --- 1.2 调用标准的 __init__ ---

  # What: 规范化输入路径。Why: 后续缓存键依赖稳定路径表示。How: 使用 resolve 保留明确异常。
  ```

## 健壮性

- 对内部不变量可用 `assert`，但需附上下文注释说明成立条件。不用于用户输入验证。
- 捕获异常当且仅当代码能添加有用上下文、安全恢复或提供有意义的回退。
- 包含可操作的错误信息，标识失败的操作和相关输入。
- 避免宽泛的异常处理，除非它记录上下文并重新抛出或返回有意为之的回退。

## 性能

- 主动标记高代价的 I/O、计算密集循环和内存密集数据结构。
- 懒加载大型数据集、外部资源和昂贵依赖。
- 大文件和数据集优先使用流式或批处理。
- 选择可能无法扩展的更简实现时，声明性能权衡。
