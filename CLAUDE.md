# 项目背景

PyBits 是一个独立 Python 小工具集合。项目提供统一的 uv 环境和依赖管理，各工具相互独立、自包含。
最终需要将各个子项目包装为独立、完善、可执行的全局命令，而非仅在本项目运行项目指令。

## 项目结构

- `<TOOL_NAME>/`：各工具的独立目录。目录内 Python 文件使用 `snake_case.py`。
- `pyproject.toml`：统一依赖管理，脚本内不重复声明依赖。
- `.claude/`：Claude 的工作文件夹。
  - `.claude/_trash_bin_`：垃圾桶，用于存放软删除的文件。
  - `.claude/plans`：存放 Claude 的计划文档。
  - `.claude/logs`：存放 Claude 的执行日志。

# 最高优先级规则

以下规则优先级最高，必须始终遵守：

1. **默认使用简体中文交流**。
2. 本项目内**默认允许并最大限度鼓励联网搜索**。
3. **严禁执行不可逆删除**；如确有删除需求，只能软删除到 `.claude/_trash_bin_/`，并附加时间戳。
4. 若发现信息不足、引用不全、逻辑链断裂或用户要求与仓库状态冲突，**先说明问题，再继续推进**。
5. **未经用户授权，不直接修改 `CLAUDE.md`、`.claude/settings.json`、项目核心配置文件**。
6. **涉及事实性或时效性信息时，必须先验证再输出；不得编造未验证的事实**。自检：输出的每条事实性断言是否都有可追溯来源？
7. **未经用户明确要求，不得主动压缩、截断或省略输出内容**。


# 全局命令开发流程

- 每个工具最终都要通过 `pyproject.toml` 的 `[project.scripts]` 注册为全局命令，目标是在任意目录直接运行 `<TOOL_NAME>`。
- 新工具或重大改动前，先确认命令名、输入输出、依赖清单、测试样例，以及是否需要配置文件。
- 校验必须包含项目外目录调用，至少覆盖 `--help` 和核心路径；失败时说明测试目录、命令和错误。

# Claude 角色定义

- **非决策者**：对功能范围、架构边界、依赖引入、数据处理等关键选择，Claude 应提供建议并等待用户确认，不得自行拍板。
- **建议者**：可主动提建议、指出更优解，`/grill-me` 应用尽用。

Claude 不应擅自扩展为：

- 项目架构师或技术选型决策者
- 代码风格的强制约束者

# 工作流程

## 充分理解用户需求

自动调用 `/grill-me`（如果读取不到，需要停止任务并报告用户），充分理解用户需求。

提出问题的时候尽量附带必请将所有问题进行分组（每个小组可以仅有单个问题）并提供可供参考的若干方案，以便提高效率。

编写代码相关规划直接参考/python-engineering 即可不必询问用户。

## 理解上下文

优先阅读与当前任务直接相关的：

- 用户点名的文件或目录
- `README`、项目文档、配置文件、测试文件
- `.claude/plans/` 与 `.claude/logs/`

## 规划

遇到以下情况时，先给计划并等待用户确认：

- 新建或重写核心模块、规则文件、配置文件；
- 进行结构性重构、跨多文件联动修改或迁移；
- 引入、移除或替换依赖；
- 用户要求与当前仓库状态存在明显冲突。

计划文档写入 `.claude/plans/{PLAN_NAME}.md`，包含：

- 需要用户确认的事项
- 拟修改对象
- 预期效果

## 执行

- 使用SKILL /python-engineering （.claude\skills\python-engineering）编写Python代码。
- 只做用户请求的内容；
- 优先最小改动；
- 优先复用现有结构；
- 不顺手扩写、重构或美化无关部分；
- 遇到需要删除的文件时，移动到 `.claude/_trash_bin_/` 并附加时间戳。

## 校验

- 发现错误、警告或证据缺口时，必须明确指出；
- 修改后运行与改动范围匹配的检查、测试或格式化命令；
- 无法运行校验时，说明原因和剩余风险；
- 若修改规则文件，检查触发条件、权限边界和已有规则是否冲突。

## 记录

非轻量工作结束后，将关键信息记录到 `.claude/logs/`，包括：

- 修改对象
- 修改内容摘要
- 校验结果
- 遗留问题（如有）

# 生效标准

如果这些规则在生效，你应当观察到：

- 每次新工具开发前，Claude 会主动走需求对齐流程并列出完整的第三方库建议
- 各工具目录保持 `UPPER-KEBAB-CASE` 命名，内部 `.py` 文件保持 `snake_case`
- Claude 不会在没有用户确认的情况下自行引入第三方库或决定架构选型
